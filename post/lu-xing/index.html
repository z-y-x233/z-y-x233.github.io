题目描述

lcllcllcl打算去n个城市旅行，他的预算是WWW

城市编号从1−n1−n1−n，城市与城市之间两两可达

城市uuu到城市vvv的花费为wu,vw_{u,v}wu,v​

lcllcllcl从城市111出发，他的目标是除了出发的城市111外，每个城市都到达并且只到达一次。在最后回到出发的城市，并且恰好用完预算，求一共有多少种不同的旅行方案

题目保证至少有一个解

分析

没啥好分析的，由于题目城市数量最大只有8，所以我们直接暴搜就完事了

解法一

因为题目要求的是从111出发并在最终回到111，所以起点和终点都为111，又因为其余每个点都只经过了一次。

那么对于任意nnn个点的路径来说起点与中点之间的所有路径就是2−n2 - n2−n的全排列

所以我们只需要枚举出2−n2 - n2−n的所有排列再计算出这条路径的花费，当当前路径花费等于 WWW 时，让最终答案加一，即可解决

那么这个时候问题来了，如何求一个序列的全排列

答案是 手写一个

next_permutation()

首先我们要知道C++有一个特别神奇的库<algorithm>​，这个库里面有很多神奇的函数

比如说我们的 next_permutation()， 他的作用是按字典序求一个序列的下一个字典序更大的排列，所以初始序列必须是字典序最小的，否则无法生成所有排列。

这个时候有人说了，没学c++不知道这个函数怎么办， 你可以手写一个啊

int next_permutation(int * start, int *end) {
	int * p2 = end - 1;
	while (p2 >= start && (*p2 < *(p2 - 1))) p2--;
	if (p2 == start) return 0;
	int * p1 = p2 - 1;
	int * p3 = end - 1;
	while (*p3 < *p1) p3--;
	swap(*p3, *p1);
	int * l = p2, *r = end - 1;
	while (l < r) 
		swap(*l++, *r--);
	return 1;
}

解法二

爆搜，用dfsdfsdfs枚举一下每次经过了哪个点，走完后判断一下是否满足条件就行，dfsdfsdfs其实就是一个枚举全排列的过程

代码
#include<bits/stdc++.h>

using namespace std;

int n, W;
int w[10][10];
int res = 0;

//参数分别为当前已经走过的点，当前花费，当前在哪个城市
void dfs(int sta, int sum, int cur) {
	//代表已经走完所有的点
	if (sta == ((1 << n) - 1)) {
		//最后需要回到第一个点
		if (sum + w[cur][1]== W) res++;
		return;
	}
	for (int i = 2; i <= n; ++i) {
		//如果当前枚举到的点没有走过那就走一下
		if ((sta >> (i - 1) & 1) == 0) {
			dfs(1 << (i - 1) | sta, sum + w[cur][i], i);
		}
	}
}

 int main() {
	 scanf("%d%d", &n, &W);
	 for (int i = 1; i <= n; ++i)
		 for (int j = 1; j <= n; ++j)
			 scanf("%d", &w[i][j]);
	 dfs(1, 0, 1);
	 printf("%d", res);
	return 0;
}
